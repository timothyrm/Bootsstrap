GUIDE FÖR BOOTRSTRAP 4

Element

Ett element är en något Default inom html, det är den som har en start tag och en slut tag, så som p och div. HTML-element kan kapslas/inneslutas, det betyder att element kan innehålla andra element. Alla HTML dokument består av kapslade HTML element. Det spelar inten roll om Det står P eller p, båda är samma när det kommer till html så liten bokstav eller stor spelar ingen roll. De viktigaste elementen är html taggen, den definerar rooten av dokumentet. Sen krävs också body för att definera dokumentets kropp. Sedan finns det också h1,h2....h6 för heading.

HTML

HTML sidorna är uppbyggda på HTML element. Med HTML-konstruktioner kan bilder och andra objekt, såsom interaktiva former anävndas i den renderade sidan. HTML ger ett sätt att skapa strukturerade dokument genom att beteckna strukturell semantik för text som rubriker, stycken, listor, länkar, citat och andra objekt. HTML-element avgränsas av taggar, skrivna med vinkelparenteser. Taggar som img och input introducerar direkt innehåll på sidan. Andra taggar som p omger och ger information om dokumenttext och kan inkludera andra taggar som underelement. Webbläsare visar inte HTML-taggarna, men använder dem för att tolka innehållet på sidan.

ID

ID används för att kunna rikta en specefik rad i ett html dokument, den är ganska lik class, bara att skillnaden är att ett id är unikt för varje dokument. Det finns ingen speciell regler för vilka element som får använda sig av id, men typiskt använder man id på större områden som exempelvis sidomenyer, behållare för artikeltext, sidhuvud, sidfot och så vidare. ID används på så sätt att du linkar en header med id och sedan länkar samma sak i navbar så kommer du skickas direkt ner till den headern om du klickar på en knapp som exempel.

Class

Class används för att länka ett class namn till stylesheeten, exempel på användning av detta kan vara att du använder dig utav div, innaför div sätter du class=container, detta gör att i stylesheeten är den länkad till .container. Class fungerar på ungefär exakt samma sätt som id men med dessa skillnader: 1. Samma class kan användas till flera element på en webbsida. 2. Du kan inte direktlänka till en class.

Meta

Meta taggen definierar metadata om HTML-dokumentet. Den är data information om data. meta taggen visas alltid i elementet head och används vanligtvis för att ange teckenuppsättning, sidbeskrivning, nyckelord, dokumentförfattare och visningsinställninga. Metadata visas inte på sidan, men dem analyseras av maskinen. Webbläsare, sökmotorer och andra webbtjänster använder metadata.

Responsive

Responsive web design är hur man får hemsidan att se snygg ut på alla olika enheter (mobil, dator osv). Responsive är hur man använder sig utav html och css för att omforma allt så det ser bra ut överallt. För att skapa en responsive hemsida så behöver man använda meta taggen till alla sidorna, ett exempel är: meta name="viewport" content="width=device-width, initial-scale=1.0". Detta gör att du kan sätta en viewport till din sida vilket ger sidan instruktiner att kunna kontrollera sidans dimensioner och storlek.

http

http står i början av en hemsida, dens uppgift är att när man tex klickar på en länk så kommer du begära ankomst vilket du får när hemsidan kommer upp. Dock är http inte så säkert då dens fokus är att visa informationen som kommer mer än hur informationen kommer. HTTP står för Hypertext Transfer Protocol, den är grunden för datakommunikation för Internet, där hypertextdokument inkluderar hyperlänkar till andra resurser som användaren lätt kan komma åt, till exempel genom att klicka med musen eller genom att trycka på skärmen i en webbläsare.

https

https är likadant som http bara att den är mer avancerad och mycket säkrare att använda. Om hemsidan begär att du ska skriva lödsenord så är det viktigt att kolla att det är https först. Det står för Hypertext Transfer Protocol Secure och är det då den inte ska kunna avlyssnas av tredje part och användaren skall kunna lita på att webbservern

CDN

CDN står för Content Distribution Network och är ett nätverk av noder på olika geografiska platser som hjälper till att distribuera innehåll. Avsikten med dessa nätverk är att tillhandahålla nättjänster nära användare, för ökad driftsäkerhet och prestanda samt minskad latens. En CDN möjliggör snabb överföring av tillgångar som behövs för att ladda internetinnehåll inklusive HTML-sidor, javaskriptfiler, formatmallar, bilder och videor.

local bootstrap

Local bootstrap är när man skickar in en link istället för laddar ner boostrap. Det är det snabbaste sättet att komma igång med bootstrap. Användingen av kommer göra allting snabbare då det är en förskriven stylesheet som hjälper en designa hemsidor mycket snabbare på ett bra sätt. Den innehåller CSS- och JavaScript-baserade designmallar för typografi, formulär, knappar, navigering och andra gränssnittskomponenter.

JQuery

JQuery är en typ av kod, såsom javascript men den tar från javascript libraryn. Den förkortar allt också så det är som "write less, do more". JQuery gör det lätt för folk att kunna göra vanliga uppgifter såsom manipulera hemsidan, svara på användarhändelser, data från servers, skapa effekter och animationer och mycket mer.

Flexbox

Flexbox är ett modernare sätt att skapa layout på webbsidor. Det är som det låter, man kan flytta runt boxar. Flexbox går att använda till väldigt mycket saker, det är väldigt simpelt att skapa en affär hemsida med det då du kan lägga upp massa bilder brevid varandra i ordning och göra så det ser snyggy ut (samma storlek osv). Det finns massa kommandon inom flexbox som gör sakerna automatiskt såsom med storlek förändring.

grid

grid eller CSS Grid Layout gör så att man kan dela stora delar eller definera storlek, position och lager mellan olika delar som är uppbygt av html. såsom tabbeller, kan grid layout låta användaren sätta in element i kolumner och rader. Men många fler layouter är antingen möjliga eller enklare med CSS-rutnät än med tabeller. Till exempel kan en rutnätbehållares underelement placera sig så att de faktiskt överlappar och lager, liknande CSS-placerade element.

code snippets

En HTML snippet är en liten mängd av source code i en HTML markup. I webbutveckling innehåller utdrag ofta HTML-kod. Ett HTML-kodavsnitt kan användas för att infoga en formaterad tabell, ett webbformulär eller ett textblock. Text editors som använder snippets kan skapa en mekanism som kommer åt katalog, och separera "utdrag" på samma sätt som textredigeraren och operativsystemet tillåter hantering av separata filer. Dessa grundläggande hanteringsförmågor inkluderar operationer som att visa, lägga till, redigera, radera, sortera, filtrera, gruppera, byta namn på och lagra utdrag i ett arkiv, katalog eller databas.
